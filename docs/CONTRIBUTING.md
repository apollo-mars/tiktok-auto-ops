# Contributing

## How to contribute
1. Pick an issue labeled `good first issue` or `help wanted`.
2. Comment to claim it.
3. Create a branch: `feat/<short-name>`.
4. Open a PR with:
   - What changed
   - How to test
   - Screenshots (if UI)

## Dev setup
- Use Docker Compose for local development.
- Keep secrets out of git; use `.env.example`.

## Code style
- TypeScript: eslint + prettier
- Python: ruff + black

## Compliance and safety
Do not implement unattended posting/login automation, fake engagement, or platform-bypass behaviors.
